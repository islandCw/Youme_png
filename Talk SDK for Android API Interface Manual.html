Talk SDK for Android API Interface Manual#

Related asynchronous / synchronous processing methods#

Youme Voice Engine SDK interface calls will return immediately. Any interface calls that require a long and time-consuming process will use asynchronous callbacks. All interfaces can be used directly in the main thread.

API call description#

API calls can be directly operated using "api." The basic flow used by the interface isinitialization->receive notification of initialization success callback->join voice channel->receive notification of successful callback to join channel->use other interfaces->leave voice channel->Deinitialization, be sure to use the interfaces exactly as described above.

Implementing callbacks#

To use the class, you need to implements the interface YouMeCallBackInterface and implement all callback functions under this interface. Callbacks are executed in child threads and cannot be used for time-consuming operations such as updating the UI.

  • First import the relevant packages:
import com.youme.voiceengine.MemberChange;
import com.youme.voiceengine.YouMeCallBackInterface;
  • Then implements the interface YouMeCallBackInterface to implement the callback method:
@Override
    public void onEvent(int eventType, int iErrorCode, String roomid, Object param){}
@Override
    public void onRequestRestAPI(int requestID, int iErrorCode, String strQuery, String strResult){}
@Override
    public void onMemberChange(String channelID, MemberChange[] changeList){}

Initialization#

  • Grammar

    int init (String strAPPKey, String strAPPSecret, int serverRegionId, String strExtServerRegionName);

  • Function
    Initialize the speech engine, do app verification and resource initialization.（Only when appkey, appsecret, and serverRegionId are the same, voice communication can be performed, and any one can be regarded as another service.）;

  • Parameter Description
    strAPPKey：App key applied from Youme background, this is the unique identifier of your application.
    strAPPSecret：Corresponds to the private key of appKey. This needs to be kept properly and not exposed to others.
    serverRegionId：Set the region code of the preferred connection server. If the region cannot be determined during initialization, you can fill in RTC_DEFAULT_SERVER, and set it later through setServerRegion. If the region code defined by YOUME_RTC_SERVER_REGION does not meet the requirements, you can set this parameter to RTC_EXT_SERVER, and then set a custom region value (such as "cn" or "ch" in China) with the parameter serverRegionName, and then customize The area value is synchronized to Youme, and we will map the server to the best area through background configuration.
    strExtServerRegionName：Custom extended server zone name. Cannot be null, can be empty string "". This parameter is valid only when the previous parameter serverRegionId is set to RTC_EXT_SERVER (otherwise it will be treated as an empty string "").

  • Return value
    Only asynchronous callback notification will be returned after YOUME_SUCCESS is returned. For other return values, please refer to YouMeErrorCode Type Definition。

  • Asynchronous callback

    //The main callback events involved are:
// YOUME_EVENT_INIT_OK  - Indicates that the initialization was successful
// YOUME_EVENT_INIT_FAILED - Indicates that the initialization failed. The most common cause of failure is a network error or an AppKey-AppSecret error.
void onEvent(int eventType, int iErrorCode, String roomid, Object param);

Voice channel management#

Join voice channel (single channel)#

  • Grammar

    int joinChannelSingleMode (String strUserID, String strRoomID, int userRole, boolean bCheckRoomExist);

  • Function
    Join voice channel (single channel mode, only one voice channel at a time).

  • Parameter Description
    strUserID: Globally unique user ID, globally refers to the scope of the current application.
    strRoomID: Globally unique channel identifier, globally means within the scope of the current application.
    userRole: The role of the user in the voice channel, see YouMeUserRole definition.
    bCheckRoomExist：Whether to check the channel to join only when it exists. The default is false: true to join when the channel exists and return an error if it does not exist (mostly used for audience roles), false to join the channel regardless of whether the channel exists.

  • Return value
    Only asynchronous callback notification will be returned after YOUME_SUCCESS is returned. For other return values, please refer to YouMeErrorCode Type Definition。

  • Asynchronous callback

    //The main callback events involved are:
//YOUME_EVENT_JOIN_OK - Successfully entered voice channel
//YOUME_EVENT_JOIN_FAILED - Failed to enter the voice channel, it may be due to a network or server problem, or the channel has not been created when bCheckRoomExist is true
void onEvent(int eventType, int iErrorCode, String roomid, Object param);

Set up whitelisted users#

  • Grammar

    int setWhiteUserList(String strChannelID,  String strWhiteUserList);

  • Function
    Set the current user's voice message reception white list, and its voice messages will only be forwarded to the white list of users. Without this interface, it will be forwarded to everyone in the channel by default.

  • Parameter Description
    strChannelID：The channel to be set (compatible with multi-channel mode, just pass the current channel in single channel mode)
    strWhiteUserList：Whitelist user list, separated by |, such as: User1 | User2 | User3; means that voice messages are only sent to User1, User2, User3. "All" means forward to everyone in the channel; "" (empty string) means not forward To any user.

  • Return value
    Only asynchronous callback notification will be returned after YOUME_SUCCESS is returned. For other return values, please refer to YouMeErrorCode Type Definition。

  • Asynchronous callback

    //The main callback events involved are:
//YOUME_EVENT_SET_WHITE_USER_LIST_OK - Successfully set up a whitelist on the specified voice channel, an exception user will return an error code YOUME_ERROR_WHITE_SOMEUSER_ABNORMAL
//YOUME_EVENT_SET_WHITE_USER_LIST_FAILED - Failed to set the whitelist on the specified voice channel, it may be due to a network or server problem
void onEvent(int eventType, int iErrorCode, String roomid, Object param);

Set whether to allow mobile network#

  • Grammar

    void setUseMobileNetworkEnabled (boolean bEnabled);

  • Function
    Set whether to allow the use of mobile networks. When both WIFI and mobile network are available, WIFI will be used preferentially. In the absence of WIFI, if the setting allows the use of mobile network, then mobile network will be used for voice communication, otherwise communication will fail.

  • Parameter Description
    bEnabled:true-mobile network is allowed, false-mobile network is prohibited.

  • Grammar

    boolean getUseMobileNetworkEnabled ();

  • Function
    Gets whether the SDK is allowed to use the mobile network for voice communication without WIFI.

  • Return value
    true-mobile network is allowed, false-mobile network is prohibited, mobile network is allowed by default.

Set whether to release the microphone device when the microphone is muted#

  • Grammar

    int setReleaseMicWhenMute(boolean enabled);

  • Function
    Set whether to release the microphone device when the microphone is muted (need to be called after the initialization is successful and before joining the room)

  • Parameter Description
    enabled： true--When the microphone is muted, release the microphone device. At this time, the third-party module is allowed to use the microphone device to record. On Android and iOS, the voice is switched from the call track to the media track, returning to the two-channel high-quality output experience (if the media volume and the call volume are different, the switch will produce a volume that becomes louder and smaller); false-- Regardless of whether the microphone is muted, the microphone device is occupied.

  • Return value
    Only asynchronous callback notification will be returned after YOUME_SUCCESS is returned. For other return values, please refer to YouMeErrorCode Type Definition。

Set whether to automatically exit the system call mode when the headset is inserted (disable the hardware signal pre-processing of the mobile phone system)#

  • Grammar

    int setExitCommModeWhenHeadsetPlugin(boolean enabled);

  • Function
    Set whether to automatically exit the system call mode when the headset is inserted (disable the hardware signal pre-processing of the mobile phone system)
    The pre-processing effects provided by the system include echo cancellation and automatic gain, which can help to suppress echo noise such as background music, reduce system resource consumption, and have no effect on Windows systems.
    Since the insertion of headphones can physically block echo generation, this effect can be set to disable to preserve the native sound quality of the background music
    Default is false, called after initialization and before joining the room

  • Parameter Description
    enabled： true--When the headset is plugged in, the system hardware signal pre-processing is automatically disabled and restored when unplugged; false--plugging and unplugging the headset is not processed.

  • Return value
    Successfully return YOUME_SUCCESS. For other return values, please refer to YouMeErrorCode Type Definition。

Set whether to enable speech volume callback#

  • Grammar

    int setMicLevelCallback(int maxLevel);

  • Function
    Set whether to enable listening to the volume callback of your own speech, and set the corresponding parameters.

  • Parameter Description
